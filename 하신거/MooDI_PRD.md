# MooDI(무디) PRD

## 1. 프로젝트 이름 / 한 줄 설명

### 프로젝트 이름

**MooDI(무디)**  
*Mood + I / AI의 합성어*

### 한 줄 설명

사용자가 현재 감정을 선택하거나 입력하면, LLM이 감정에 맞는 음악 추천 방향을 생성하고 YouTube API를 통해 음악을 추천하며, 저장한 곡을 라이브러리에서 관리하고 가벼운 음악 믹스 기능까지 제공하는 **감정 기반 개인화 음악 추천 웹 서비스**이다.

---

## 2. 목표: 왜 만드나

MooDI는 사용자가 직접 곡명이나 가수를 검색하지 않아도, 자신의 현재 감정에 맞는 음악을 빠르게 추천받고 저장할 수 있도록 돕는 서비스이다.

기존 음악 플랫폼은 사용자가 원하는 음악을 직접 검색하거나, 플랫폼이 미리 정한 알고리즘 추천을 받아야 한다. 하지만 사용자는 항상 “정확히 어떤 노래를 듣고 싶은지” 알고 있는 것이 아니라, “지금 내 기분에 맞는 음악”을 듣고 싶을 때가 많다.

MooDI는 이 문제를 해결하기 위해 다음 목표를 가진다.

1. 사용자가 현재 감정을 버튼으로 빠르게 선택할 수 있게 한다.
2. 사용자가 직접 감정을 문장으로 입력해 더 구체적인 추천을 받을 수 있게 한다.
3. LLM을 활용해 감정에 맞는 음악 추천 방향, 검색 키워드, 추천 이유를 생성한다.
4. YouTube API를 활용해 실제 음악 영상 데이터를 가져온다.
5. 사용자가 마음에 드는 음악을 개인 라이브러리에 저장할 수 있게 한다.
6. 저장한 음악은 라이브러리에서 다시 확인하거나 삭제할 수 있게 한다.
7. 네이버 로그인을 통해 사용자별 라이브러리와 프로필을 관리한다.
8. 뮤직 믹스 기능을 통해 가벼운 노래 편곡 또는 분위기 변환 기능을 제공한다.
9. 완성된 웹 서비스를 Cloudflare에 배포해 외부에서도 접속 가능하게 한다.

## 3. 사용자 입력

MooDI에서 사용자가 입력하거나 선택하는 정보는 다음과 같다.

| 입력 항목 | 입력 방식 | 예시 |
|---|---|---|
| 로그인 정보 | 네이버 로그인 버튼 | 네이버 계정으로 로그인 |
| 감정 빠른 선택 | 감정 버튼 클릭 | 기쁨, 슬픔, 화남, 피곤함, 설렘 |
| 감정 직접 입력 | 하단 입력창에 텍스트 입력 | “오늘 너무 지쳤어”, “신나는 노래 듣고 싶어” |
| 메뉴 선택 | 좌측 햄버거 메뉴 클릭 후 선택 | 홈, 뮤직 믹스, 라이브러리 |
| 음악 저장 | 추천 음악 카드의 저장 버튼 클릭 | 마음에 드는 노래 저장 |
| 음악 삭제 | 라이브러리의 삭제 버튼 클릭 | 저장한 노래 삭제 |
| 프로필 메뉴 | 우측 상단 프로필 키 클릭 | 프로필 변경, 설정 변경, 로그아웃 |
| 뮤직 믹스 설정 | 뮤직 믹스 화면에서 선택 | 더 잔잔하게, 더 신나게, lo-fi 느낌으로 |
| 편곡 강도 | 버튼 또는 슬라이더 | 약하게 / 보통 / 강하게 |

## 4. 외부 데이터 / API: 무엇을 가져오나

MooDI는 실제 구현을 목표로 하며, 다음 외부 데이터와 API를 활용한다.

### 4-1. 네이버 로그인 API

| 항목 | 내용 |
|---|---|
| API 이름 | 네이버 로그인 API |
| 사용 목적 | 사용자 로그인 및 사용자 식별 |
| 가져오는 정보 | 이름, 이메일, 프로필 이미지 또는 닉네임 |
| 서비스 내 활용 | 프로필 표시, 사용자별 라이브러리 저장, 설정 관리 |

### 사용 흐름

```text
사용자가 네이버 로그인 클릭
→ 네이버 인증 페이지 이동
→ 로그인 성공
→ 사용자 정보 가져오기
→ MooDI 프로필에 표시
→ 사용자별 라이브러리 사용 가능
```

### 4-2. YouTube Data API

| 항목 | 내용 |
|---|---|
| API 이름 | YouTube Data API |
| 사용 목적 | 음악 검색 및 영상 정보 가져오기 |
| 가져오는 정보 | 영상 제목, 채널명, 썸네일, 영상 ID, 영상 URL |
| 서비스 내 활용 | 추천 음악 카드 출력, 라이브러리 저장, 음악 재생 연결 |

### 사용 흐름

```text
LLM이 추천 검색어 생성
→ YouTube API에 검색 요청
→ 음악 영상 목록 가져오기
→ 추천 카드 형태로 화면에 출력
```

### 4-3. LLM API

| 항목 | 내용 |
|---|---|
| API 이름 | OpenAI API 또는 기타 LLM API |
| 사용 목적 | 감정 분석, 음악 추천 검색어 생성, 추천 이유 생성 |
| 입력 데이터 | 사용자의 감정 버튼 값 또는 직접 입력 문장 |
| 출력 데이터 | 추천 분위기, YouTube 검색 키워드, 추천 문장, 추천 이유 |
| 서비스 내 활용 | 단순 키워드 매핑보다 자연스러운 개인화 추천 제공 |

### 사용 흐름

```text
사용자 입력: “오늘 너무 지치고 아무것도 하기 싫어”
→ LLM 분석
→ 감정: 피곤함 / 우울함
→ 추천 분위기: 잔잔한, 위로되는, 낮은 템포
→ YouTube 검색어: “calm korean lo-fi playlist for tired mood”
→ 추천 문장 생성
→ YouTube API 검색
```

|---|---|

## 5. 처리 규칙: 매핑

MooDI의 핵심 처리 규칙은 다음과 같다.

```text
사용자 감정 입력
→ LLM이 감정 분석
→ LLM이 음악 추천 방향 생성
→ LLM이 YouTube 검색어 생성
→ YouTube API로 음악 검색
→ 추천 음악 카드 출력
→ 사용자가 저장하면 라이브러리에 저장
```

### 5-1. 확정 감정 버튼 5개

입력창 위에는 사용자가 빠르게 선택할 수 있는 감정 버튼 5개를 배치한다.

| 감정 버튼 | 의미 | LLM에 전달되는 기본 프롬프트 |
|---|---|---|
| 기쁨 | 밝고 즐거운 상태 | 사용자가 기쁜 기분일 때 어울리는 음악을 추천해줘 |
| 슬픔 | 우울하거나 가라앉은 상태 | 사용자가 슬픈 기분일 때 위로가 되는 음악을 추천해줘 |
| 화남 | 스트레스나 분노가 있는 상태 | 사용자가 화가 났을 때 감정을 해소할 수 있는 음악을 추천해줘 |
| 피곤함 | 지치고 쉬고 싶은 상태 | 사용자가 피곤할 때 편하게 들을 수 있는 음악을 추천해줘 |
| 설렘 | 기대감, 두근거림이 있는 상태 | 사용자가 설레는 기분일 때 어울리는 음악을 추천해줘 |

### 5-2. 감정 버튼 클릭 시 처리 규칙

```text
사용자가 “피곤함” 버튼 클릭
→ 입력값을 “피곤함”으로 저장
→ LLM에게 “피곤한 기분에 어울리는 음악 추천 방향” 요청
→ LLM이 추천 분위기, 검색어, 추천 문장 생성
→ 생성된 검색어로 YouTube API 검색
→ 검색 결과를 음악 카드로 출력
```

### 5-3. 직접 입력 시 처리 규칙

사용자가 직접 문장을 입력한 경우, LLM이 문장의 감정을 분석한다.

| 사용자 입력 예시 | LLM 분석 결과 | 추천 방향 |
|---|---|---|
| 오늘 너무 지쳤어 | 피곤함 | 낮은 템포, 잔잔한, lo-fi |
| 신나는 노래 듣고 싶어 | 기쁨 | 밝은, 빠른 템포, pop |
| 화가 나서 스트레스 풀고 싶어 | 화남 | 강한 비트, rock, hip-hop |
| 우울하고 슬퍼 | 슬픔 | 위로되는, 발라드, 어쿠스틱 |
| 썸 타는 느낌의 노래 듣고 싶어 | 설렘 | 몽환적인, 인디팝, R&B |

### 5-4. LLM 추천 출력 규칙

LLM은 다음 형식으로 결과를 생성한다.

```json
{
  "emotion": "피곤함",
  "mood": ["잔잔한", "편안한", "낮은 템포"],
  "youtubeSearchQuery": "calm lo-fi playlist for tired mood",
  "recommendationText": "오늘은 힘을 빼고 편하게 들을 수 있는 잔잔한 음악이 어울려요.",
  "reason": "피곤한 상태에서는 빠르고 자극적인 음악보다 반복적이고 부드러운 사운드가 부담을 줄여줄 수 있습니다."
}
```

### 5-5. 화면 이동 규칙

좌측 상단의 슬래시 3개, 즉 햄버거 메뉴를 누르면 사이드 메뉴가 열린다.

| 메뉴 | 이동 화면 | 주요 기능 |
|---|---|---|
| Home | 홈 화면 | 감정 선택, 감정 입력, 추천 음악 확인 |
| Music Mix / 뮤직 믹스 | 뮤직 믹스 화면 | 선택한 노래를 가볍게 편곡하거나 분위기 변환 |
| Library / 라이브러리 | 라이브러리 화면 | 저장한 YouTube 음악 목록 확인 및 삭제 |

### 5-6. 프로필 키 처리 규칙

오른쪽 위의 프로필 키를 누르면 프로필 관련 메뉴가 열린다.

| 메뉴 | 기능 |
|---|---|
| 프로필 변경 | 닉네임, 프로필 이미지 변경 |
| 설정 변경 | 테마, 추천 방식, 계정 설정 변경 |
| 로그아웃 | 현재 로그인된 네이버 계정 로그아웃 |

### 5-7. 라이브러리 저장 규칙

```text
사용자가 추천 음악 카드의 “저장” 버튼 클릭
→ 로그인 여부 확인
→ 로그인 상태이면 YouTube 영상 정보를 라이브러리에 저장
→ 저장 완료 메시지 출력
→ Library 화면에서 저장한 노래 확인 가능
```

저장되는 정보는 다음과 같다.

| 저장 항목 | 예시 |
|---|---|
| 사용자 ID | naver_123456 |
| 영상 제목 | Calm Lo-fi Study Music |
| 채널명 | Lo-fi Channel |
| 썸네일 | YouTube 썸네일 URL |
| 영상 URL | https://youtube.com/... |
| 감정 태그 | 피곤함 |
| 저장 날짜 | 2026-05-26 |

### 5-8. 라이브러리 삭제 규칙

```text
사용자가 Library 화면에서 저장된 노래의 “삭제” 버튼 클릭
→ 삭제 확인 메시지 출력
→ 사용자가 확인 클릭
→ 해당 노래를 라이브러리에서 삭제
→ 삭제 완료 메시지 출력
```

출력 문구:

```text
라이브러리에서 삭제되었습니다.
```

### 5-9. 뮤직 믹스 처리 규칙

뮤직 믹스는 사용자가 선택한 노래를 기반으로 **가벼운 편곡 방향**을 제안하거나, 음악의 분위기를 바꾸는 기능이다.

초기 구현에서는 실제 음원 파일을 직접 편집하기보다, 다음 방식으로 구현한다.

1. 사용자가 추천 음악 또는 라이브러리 음악 중 하나를 선택한다.
2. 편곡 방향을 선택한다.
3. LLM이 해당 노래를 어떤 분위기로 바꿀지 설명한다.
4. 필요하면 YouTube API로 비슷한 리믹스/커버/lo-fi 버전 음악을 검색한다.
5. 결과를 “MooDI Mix” 카드로 보여준다.

### 5-10. 뮤직 믹스 선택 옵션

| 옵션 | 설명 | 예시 출력 |
|---|---|---|
| 더 잔잔하게 | 템포를 낮추고 부드러운 분위기로 변환 | lo-fi, acoustic, piano 느낌 |
| 더 신나게 | 비트와 에너지를 높이는 방향 | upbeat, dance, pop remix 느낌 |
| 더 몽환적으로 | 공간감 있고 감성적인 분위기로 변환 | dream pop, synth, ambient 느낌 |
| 집중용으로 | 가사나 강한 비트를 줄인 방향 | instrumental, study, focus 느낌 |
| 위로받는 느낌으로 | 차분하고 감성적인 방향 | ballad, soft vocal, healing 느낌 |

### 5-11. 뮤직 믹스 출력 예시

```text
[MooDI Mix]

원곡:
Calm Lo-fi Study Music

선택한 편곡 방향:
더 몽환적으로

AI 편곡 제안:
이 곡은 기존의 잔잔한 lo-fi 분위기를 유지하면서, 신스 패드와 공간감 있는 리버브를 추가하면 더 몽환적인 분위기로 바꿀 수 있습니다.

추천 검색 결과:
1. dreamy lo-fi remix playlist
2. ambient lo-fi night mix
3. synthwave calm study music
```

## 6. 출력 형식

MooDI의 출력은 웹 화면 기준으로 구성한다.

### 6-1. 기본 홈 화면

기본 화면은 사용자가 감정을 입력하거나 선택하는 중심 화면이다.

```text
[상단]
왼쪽: 햄버거 메뉴 버튼
중앙: MooDI 로고
오른쪽: 프로필 키 버튼

[중앙]
넓은 여백 중심의 깔끔한 화면

[감성 타이틀]
음악적 감성을 살린 원형 타이틀 배치
- “MooDI” 또는 “moody” 텍스트를 음악 로고처럼 변형
- 원형 안에 텍스트를 넣어 감성적인 브랜드 심볼처럼 사용
- 예: o를 음표, CD, LP, 파형 형태로 변형
- 기본 홈 화면의 시각적 중심 요소로 사용

[입력창 위]
감정 빠른 선택 버튼 5개
- 기쁨
- 슬픔
- 화남
```

### 6-2. 감성 타이틀 디자인 방향

메인 화면의 중앙에는 단순 텍스트 로고 대신, 음악 서비스의 분위기를 보여주는 **원형 감성 타이틀**을 배치한다.

#### 디자인 의도

- 사용자가 첫 화면에서 “음악 서비스”라는 인상을 바로 받을 수 있게 한다.
- MooDI의 감정 기반 추천 서비스라는 느낌을 부드럽고 감성적으로 전달한다.
- 기능 중심의 딱딱한 화면이 아니라, 음악 플레이리스트나 앨범 커버 같은 분위기를 만든다.

#### 디자인 구성

| 요소 | 설명 |
|---|---|
| 원형 프레임 | LP, CD, 플레이 버튼, 사운드 웨이브를 연상시키는 원형 배치 |
| 텍스트 | MooDI 또는 moody를 감성적으로 변형한 타이포그래피 |
| 음악 요소 | 음표, 파형, 작은 별, 빛 번짐, 그라데이션 등을 최소한으로 사용 |
| 색상 | 다크 배경 위에 보라, 민트, 블루 계열 포인트 컬러 사용 |
| 위치 | 메인 화면 중앙, 감정 버튼 위쪽 |

#### 예시 문구

```text
메인 화면 중앙에는 MooDI 로고를 음악적 감성으로 변형한 원형 타이틀을 배치한다.
MooDI의 알파벳 일부를 음표, CD, 파형처럼 변형하여 사용자가 첫 화면에서 음악 서비스의 분위기를 느낄 수 있게 한다.
```

## 7. 예외처리 계획

프로그램이 정상적으로 동작하지 않는 상황을 대비해 다음 예외처리를 준비한다.

### 7-1. 로그인하지 않은 상태에서 라이브러리 저장 시도

**상황**  
사용자가 로그인하지 않은 상태에서 노래를 저장하려고 하는 경우.

**처리 계획**

- 저장 기능을 실행하지 않는다.
- 네이버 로그인 안내 메시지를 출력한다.
- 로그인 버튼을 제공한다.

**출력 문구**

```text
노래를 저장하려면 로그인이 필요합니다. 네이버로 로그인해주세요.
```

### 7-2. 잘못된 감정 입력

**상황**  
사용자가 감정과 관련 없는 문장을 입력하거나, LLM이 감정을 분석할 수 없는 경우.

**예시**

```text
“감자튀김 먹고 싶다”
```

**처리 계획**

- 감정을 자동 분류하지 못했다는 메시지를 출력한다.
- 감정 버튼 5개 중 하나를 선택하도록 안내한다.
- 기본 추천값은 사용하지 않고 사용자 선택을 기다린다.

**출력 문구**

```text
감정을 정확히 파악하기 어려워요. 기쁨, 슬픔, 화남, 피곤함, 설렘 중 하나를 선택해주세요.
```

### 7-3. LLM API 호출 실패

**상황**  
LLM API 요청이 실패하거나 응답이 오지 않는 경우.

**원인 예시**

- API 키 오류
- 인터넷 연결 문제
- 요청 횟수 제한 초과
- 서버 응답 지연

**처리 계획**

- 기술적인 오류 코드를 그대로 보여주지 않는다.
- 감정 버튼별 기본 검색어를 임시로 사용한다.
- 임시 추천이라는 안내 문구를 보여준다.

**출력 문구**

```text
AI 추천을 불러오지 못했습니다. 기본 추천 방식으로 음악을 찾아드릴게요.
```

### 7-4. YouTube API 호출 실패

**상황**  
YouTube API 요청이 실패하거나 응답이 오지 않는 경우.

**처리 계획**

- 사용자에게 기술적인 오류 코드를 그대로 보여주지 않는다.
- 미리 준비된 더미 음악 데이터를 대신 보여준다.
- “임시 추천 목록”임을 안내한다.

**출력 문구**

```text
YouTube 음악 정보를 불러오지 못했습니다. 임시 추천 목록을 보여드릴게요.
```

### 7-5. 네이버 로그인 실패

**상황**  
네이버 로그인 인증이 실패하거나 사용자가 로그인을 취소한 경우.

**처리 계획**

- 로그인 실패 메시지를 출력한다.
- 다시 로그인할 수 있는 버튼을 제공한다.
- 로그인하지 않아도 홈 화면과 감정 추천 기능은 사용할 수 있게 한다.
- 단, 라이브러리 저장 기능은 제한한다.

**출력 문구**

```text
네이버 로그인이 완료되지 않았습니다. 다시 시도해주세요.
```

### 7-6. 라이브러리 데이터 저장 실패

**상황**  
사용자가 저장 버튼을 눌렀지만 DB 또는 JSON 저장 중 오류가 발생한 경우.

**처리 계획**

- 저장 실패 메시지를 출력한다.
- 같은 노래를 다시 저장할 수 있도록 버튼을 유지한다.
- 이미 저장된 노래인지도 함께 확인한다.

**출력 문구**

```text
노래를 저장하는 중 문제가 발생했습니다. 잠시 후 다시 시도해주세요.
```

### 7-7. 중복 저장 시도

**상황**  
사용자가 이미 저장한 노래를 다시 저장하려고 하는 경우.

**처리 계획**

- 중복 저장하지 않는다.
- 이미 라이브러리에 있다는 안내 문구를 출력한다.

**출력 문구**

```text
이미 라이브러리에 저장된 노래입니다.
```

### 7-8. 라이브러리 삭제 실패

**상황**  
사용자가 삭제 버튼을 눌렀지만 데이터 삭제 중 오류가 발생한 경우.

**처리 계획**

- 삭제 실패 메시지를 출력한다.
- 다시 삭제할 수 있도록 버튼을 유지한다.
- 실제 데이터가 삭제되었는지 확인 후 화면을 갱신한다.

**출력 문구**

```text
노래를 삭제하는 중 문제가 발생했습니다. 잠시 후 다시 시도해주세요.
```

### 7-9. Cloudflare 배포 실패

**상황**  
Cloudflare Pages 또는 Cloudflare Tunnel 배포 과정에서 오류가 발생한 경우.

**원인 예시**

- 빌드 오류
- 환경변수 누락
- API 키 미설정
- 라우팅 문제

**처리 계획**

- 로컬 실행 버전을 먼저 확인한다.
- 배포 로그를 확인한다.
- API 키와 환경변수를 다시 설정한다.
- 배포 실패 시 로컬 시연을 대체 방법으로 준비한다.

**출력 문구**

```text
배포 중 문제가 발생했습니다. 로컬 실행 상태를 먼저 확인해주세요.
```

## 최종 요약

MooDI는 사용자가 감정 버튼을 클릭하거나 감정을 직접 입력하면, LLM이 사용자의 감정을 분석하고 음악 추천 방향과 YouTube 검색어를 생성한 뒤, YouTube API를 통해 실제 음악 영상을 추천하는 감정 기반 음악 추천 웹 서비스이다.

기본 화면은 중앙의 MooDI 로고와 감정 입력창을 중심으로 구성되며, 입력창 위에는 **기쁨, 슬픔, 화남, 피곤함, 설렘**의 빠른 감정 선택 버튼이 배치된다.

메인 화면 중앙에는 **MooDI/moody를 음악적 감성으로 변형한 원형 타이틀**을 배치한다. 이 타이틀은 LP, CD, 파형, 음표를 연상시키는 원형 로고처럼 구성하여, 사용자가 첫 화면에서 음악 서비스의 분위기를 직관적으로 느낄 수 있게 한다.

좌측 상단의 햄버거 메뉴를 누르면 **Home, Music Mix, Library** 메뉴가 나타난다. Library에서는 사용자가 저장한 YouTube 음악을 확인하고 삭제할 수 있으며, Music Mix에서는 선택한 곡을 더 잔잔하게, 더 신나게, 더 몽환적으로 바꾸는 등 가벼운 편곡 방향을 제안받을 수 있다.

오른쪽 위의 프로필 키를 누르면 프로필 변경, 설정 변경, 로그아웃 기능을 사용할 수 있고, 네이버 로그인을 통해 사용자별 라이브러리 저장 기능을 제공한다.

완성된 서비스는 Cloudflare Pages 또는 Cloudflare Tunnel을 통해 배포하여 팀원과 교수자가 URL로 접속할 수 있게 한다.
